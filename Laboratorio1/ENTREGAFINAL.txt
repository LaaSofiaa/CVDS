1. Averigua para qué sirve y como se usan estos comandos git add y git commit -m “mensaje”

-JAIDER:
Git Add: se utiliza para añadir cambios o archivos al repositorio.
Git commit: hace un respaldo de los cambios hechos en nuestro repositorio en cierto punto.

-LAURA:
Git add:   Este comando añade cambios o archivos desde directorio de trabajo al repositorio local. 
Git commit -m: Este comando permite capturar y guardar de manera permanente los cambios preparados al momento del proyecto en un historial de este.

TRABAJO DE PAREJA

2. Owner y Colaborador editan el archivo README.md al mismo tiempo e intentan subir los cambios al mismo tiempo. ¿Qué sucedió?

Lo que sucedió fue que se subió información diferente en los documentos de la misma rama, por ende, hubo conflictos y salen signos de este estilo ( <<< === y >>> ) con código de identificación, que se utilizan para solucionar el problema.  Para resolver los conflictos, el colaborador debe primero hacer pull, consecutivamente corregir manualmente el archivo, finalizando con el push. 

3. ¿Hay una mejor forma de trabajar con git para no tener conflictos?

Tomar una versión inicial donde se obtenga una rama principal y ramas de trabajo donde en cada una de ellas se manipule un archivo diferente para no tener conflictos de merge, haciendo commits frecuentes, utilizando el pull y push.

4. ¿Qué es y cómo funciona el Pull Request?

Para solicitar y proponer modificaciones a los owners del proyecto se puede utilizar un pull request, y para ver el funcionamiento se crea ramas diferentes a la principal y realizar cambios ahí, luego se hace la solitud del pull request y el owner revisa cada fragmento que se subió, para luego fusionar el código.
